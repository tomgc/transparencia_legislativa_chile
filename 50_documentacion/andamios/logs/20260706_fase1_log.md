## 3. Cambios sustantivos (que, por que, verificacion)

- **Scaffold manual (no via scaffold_proyecto.R).** El directorio raiz ya
  existia (con `.claude/`), y `scaffold_proyecto.R` aborta ante carpetas
  existentes. Se replicaron las plantillas canonicas a mano (escaner y protocolo
  copiados literal desde `herramientas_dev/`). Verificado: escaner corre,
  estructura por decenas correcta.
- **Cliente HTTP unico `descargar_xml_camara()`** con backoff exponencial,
  timeout y User-Agent (POLITICA 5.3.9). Motivo: toda extraccion pasa por un
  punto con resiliencia ante la fuente externa.
- **Cache `con_cache()`** date-stamped en `20_insumos/camara/`: idempotencia y
  cortesia con la API (POLITICA 5.2.3). La segunda corrida reutiliza la captura.
- **Consolidacion con claves ordenadas + UTF-8 explicito** (`enc2utf8` +
  `useBytes`). Verificado: sin mojibake, tildes/ñ intactas.

## 5. Bugs encontrados y resueltos (durante la construccion)

- **Seleccion de militancia vigente.** Primer intento asumio `FechaTermino`
  vacia para la militancia actual; ninguna lo esta (todas cierran en fin de
  periodo). Causa raiz: supuesto sobre la forma del dato. Fix: militancia de
  mayor `FechaInicio`. Verificado contra crudo (18 partidos, suma 155).
- **Dominio de OpcionVoto incompleto.** Se asumio {0,1,2,4}; la API trae
  tambien `3` (Dispensado). El validador lo detecto con WARN (no fallo en
  silencio, POLITICA 5.3.8). Fix: se agrego `3=dispensado` a `DOMINIO_VOTO` y
  se re-derivo desde cache (sin refetch). Verificado: 0 valores fuera de dominio.
- **rol autor/coautor inexistente.** Se asumio jerarquia por `Orden`; es
  siempre 0. Fix: rol uniforme `firmante`, documentado. Verificado: no se
  fabrica jerarquia.
